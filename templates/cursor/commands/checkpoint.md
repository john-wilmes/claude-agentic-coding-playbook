# Checkpoint

Save all session state and prepare for a clean handoff.

1. **Update memory/notes**: Record what was accomplished, non-obvious discoveries, and next steps. Always update the "Current Work" section (replace previous entry, not append).
2. **Run quality gates**: Type-check, lint, and test if configured. Document failures rather than blocking.
3. **Commit and push**: Stage relevant files (not build artifacts or secrets), commit with a descriptive message, and push.
4. **Report**: Summarize what was committed and recommend starting a fresh session for the next task.
